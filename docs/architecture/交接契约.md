# Handoff Contract

## Purpose

角色之间禁止只靠口头聊天切换上下文。
所有阶段切换必须通过结构化 handoff 完成，减少上下文漂移与 token 浪费。

## Planner → Developer

### Required fields
- task_id
- task_type
- requirement_summary
- scope
- non_scope
- touched_modules
- constraints
- implementation_plan
- acceptance_criteria
- docs_updated
- open_questions
- risks

### Rules
- 如果 docs 未更新，要明确说明为什么无需更新
- implementation_plan 必须是 developer 可执行的步骤，而不是泛泛建议
- acceptance_criteria 必须可验证

## Developer → Ops

### Required fields
- task_id
- code_changes
- files_changed
- whitebox_tests_added
- unit_tests_added
- local_validation_commands
- local_validation_results
- known_issues
- deployment_notes
- rollback_notes

### Rules
- 不允许只写“已完成，请测试”
- 必须写明具体改了什么、如何验证、还有哪些已知边界
- 如果测试未补齐，必须明确标红为 blocker 或 risk

## Ops → Summary

### Required fields
- task_id
- acceptance_result
- test_review_result
- deployment_result
- blockers
- followups
- final_summary

## Storage recommendation

建议每个 run 使用独立目录，例如：

- `runs/<task-id>/规划交接模板.md`
- `runs/<task-id>/开发交接模板.md`
- `runs/<task-id>/ops-summary.md`

长期状态以 artifacts 为准，不以聊天历史为准。
