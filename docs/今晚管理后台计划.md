# Admin 今晚短计划（2026-03-17）

目标：今晚只收 3 件事，短平快，不扩 scope。

## 1. 后台进一步美化拆分

### 今晚要做
- 把 admin 视觉层拆成 3 个稳定块：
  1. 顶部概览区（标题 / 当前环境 / 关键说明）
  2. 页面内操作区（筛选 / 表单 / CTA）
  3. 列表展示区（桌面表格 + 移动卡片）
- 抽一层可复用 admin UI 组件或样式约定，优先覆盖：
  - section header
  - filter bar
  - feedback state（loading / empty / error / success）
  - mobile card / desktop table 容器
- 先统一 `admin` 首页、用户页、币种页，不新开更多后台页面。

### 今晚完成标准
- 三个 admin 页面视觉层级一致
- 手机端不靠横向滚动完成主要操作
- 页面不再混杂“hero / 表单 / 表格”不同风格写法

## 2. 审计日志页最小范围

### 今晚只做最小版
- 新增 admin 审计日志页路由与导航入口
- 后端若已有查询能力就直接接；若还没有，则先补最小只读列表接口：
  - `GET /api/admin/audit-logs`
- 首版只展示最近日志，不做复杂筛选，不做详情抽屉，不做导出
- 首批字段控制在：
  - time
  - actor
  - action
  - target_type
  - target_id
  - summary

### 今晚完成标准
- 管理员能在后台看到最近审计记录
- 至少能覆盖用户角色/状态变更、币种配置变更两类动作
- 页面支持桌面列表 + 移动卡片两种阅读方式

## 3. Admin 冒烟测试验收标准

### 今晚先定验收，不追求大而全自动化
按“能登录、能进入、能查看、能修改、能留痕”验收：

1. 使用管理员账号登录成功
2. `/admin` 可访问，普通用户不可进入 admin 变更页
3. 用户页可拉到列表，并成功修改 1 个用户 role 或 status
4. 币种页可拉到列表，并成功新增或修改 1 条 symbol config
5. 审计日志页可看到上面两类管理动作至少各 1 条
6. 手机宽度下，用户页 / 币种页 / 审计页都可读、可点、无致命布局错乱
7. 关键失败场景有明确反馈：未授权、加载失败、保存失败

### 建议执行顺序
1. 先拆 admin UI 层，统一壳子
2. 再补审计日志最小页
3. 最后按上述 7 条跑一轮冒烟并记录结果

## 明确不做
- 不做审计日志高级筛选/搜索
- 不做审计详情页
- 不做批量操作
- 不做新的权限模型
- 不做生产初始化/真实 env 调整
