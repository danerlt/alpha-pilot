# 2026-03-17 Runtime Config Plan

## 新需求

用户希望：

1. 前端可以配置 Binance 测试盘与实盘 API Key
2. 前端可以切换测试盘 / 实盘
3. 这些配置持久化存储，并且敏感字段需要加密存储
4. 数据库中的配置更新后，进程内运行时配置立即热更新
5. 一部分目前放在 `.env` 的业务配置迁移到数据库；`.env` 仅保留数据库、Redis、加密主密钥等基础设施级配置

## 影响范围

这是一个跨层需求，涉及：

- 数据模型与 Alembic migration
- 配置读取架构（env-only → env + db runtime overrides）
- 敏感信息加密
- 运行时热更新与缓存失效
- 后端配置 API
- 前端配置面板

## 排期拆分

### Chunk 1 — 后端配置底座
- 新增数据库配置表
- 新增加密/解密服务
- 新增运行时配置管理器
- 让 `get_settings()` 支持 env + db override
- 当配置更新时，清理 Binance client 等缓存

### Chunk 2 — 后端配置 API
- 查询当前运行时配置
- 更新测试盘 / 实盘 API Key
- 更新当前 trading mode
- 更新部分风险参数 / LLM 配置
- 更新后立即热刷新进程内配置

### Chunk 3 — 前端配置面板
- 新增配置卡片/页面
- 支持输入测试盘 / 实盘 API Key
- 支持切换模式
- 对 mainnet 提供明显风险提示
- 只展示密钥是否已配置，不回显明文

### Chunk 4 — 配置迁移与收口
- 梳理哪些配置继续保留在 `.env`
- 哪些迁移到数据库
- 文档更新
- 补回归测试

## 安全边界

- 不在前端回显 API Secret 明文
- 数据库只存加密后的敏感字段
- `.env` 仍需保留基础设施级配置与加密主密钥
- 第一轮不改变交易执行语义，只改变“配置来源与切换机制”
