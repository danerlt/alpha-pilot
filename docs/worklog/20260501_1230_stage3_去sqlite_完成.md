## 关键设计变更

### 测试库使用模式

老板指令"测试库用一个本地的新库"已落实：

- 业务运行：`alphapilot` 库（DEV / PROD）
- 测试运行：`alphapilot_test` 库（自动建库 + alembic upgrade）
- 隔离：测试间 TRUNCATE 所有用户表，保留 schema

启动序列（pytest 第一次跑）：
1. `_ensure_test_database()`：连 admin DB 检查 `alphapilot_test` 是否存在，不存在则 CREATE
2. `subprocess` 跑 `alembic upgrade head`（DATABASE_URL=test url）
3. `os.environ["TEST_DATABASE_URL"] = test_url` 让所有测试 fixture 用此 URL
4. 后续每测试结束 TRUNCATE 所有表
