## Stage 5 详细

### 进程拆分
- `scripts/start_api.py`：API 进程入口（uvicorn 多 worker，UVICORN_WORKER_NUM 环境变量）
- `scripts/start_scheduler.py`：scheduler 进程入口（APScheduler + EventShuttle daemon + 任务队列消费）

### 新增 src/schedulers/
- `strategy_pipeline_scanner.py`：APScheduler 周期触发策略循环
- `position_monitor_scanner.py`：APScheduler 周期触发持仓监控
- `event_shuttle.py`：daemon thread loop（outbox → Redis Pub/Sub）

### lifespan 解耦
- `src/app.py` lifespan **不再启动 APScheduler**
- 向后兼容：`ALPHAPILOT_API_EMBED_SCHEDULER=1` 可强开内嵌（开发用）

### docker-compose 改造
- 新增 `api` service（多 worker）+ `scheduler` service（单实例 + stop_grace_period: 60s）
- 旧 `backend` service 转 `legacy` profile（兼容旧 deploy 脚本）

### graceful shutdown
- SIGTERM → stop_flag.set()
- APScheduler `shutdown(wait=True)` 等待运行中 job 完成
- EventShuttle daemon `join(timeout=5)` 5 秒优雅窗口
- Docker `stop_grace_period: 60s` 兜底，超时 SIGKILL → 下次启动 recover_orphan_tasks 兜底
