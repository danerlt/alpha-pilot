# V0.1 老 worker / service / 测试整体淘汰

日期：2026-05-01
分支：refactor/cleanup-twin-files

## 做了什么

把仓库里 V0.1 函数式老实现和 V1 类式新实现并行的 8 对 "双胞胎" 一次性淘汰，scheduler 完全切到 V1 (`new_*_job`)，并去掉 `USE_NEW_PIPELINE_WORKER` feature flag。

### 删除文件

V0.1 老 worker (2)：
- `backend/src/workers/strategy_loop.py`
- `backend/src/workers/position_monitor.py`

V0.1 老 service (8)：
- `backend/src/services/execution/account_state_service.py`
- `backend/src/services/execution/execution_guard_service.py`
- `backend/src/services/execution/candle_service.py`
- `backend/src/services/execution/order_execution_service.py`
- `backend/src/services/execution/monitoring_service.py`
- `backend/src/services/insight/experience_store.py`
- `backend/src/services/insight/indicators_calculator.py`
- `backend/src/services/insight/regime_classifier.py`

V0.1 专属测试 (2)：
- `backend/tests/unit/test_guard.py`
- `backend/tests/unit/test_indicators.py`

### Scheduler 切换

`backend/src/schedulers/strategy_pipeline_scanner.py` 与 `position_monitor_scanner.py` 现在直接转发到 `src.workers.scheduler_jobs.new_strategy_pipeline_job` / `new_position_monitor_job`，不再 import V0.1 worker，KillSwitch / risk_profile / adapter / LLM 装配全部由 V1 job 内部统一处理。

`backend/src/app.py` lifespan 移除 `_strategy_job` / `_monitor_job` 旧路径以及 `USE_NEW_PIPELINE_WORKER` 分支，embedded scheduler 一律走 V1 job。

### Feature flag 清理

- `src/configs/app_configs.py` 删除 `USE_NEW_PIPELINE_WORKER` 字段
- `example.env` 删除对应配置注释

## 为什么

V0.1 与 V1 长期并行带来重复维护成本；Plan 5 已稳定运行，flag 长期 `true`，旧路径变死代码。

## 如何验证

`cd backend && ALPHAPILOT_SKIP_SECRET_VALIDATION=1 uv run pytest -q --tb=short`：

- 之前：~480 passed
- 之后：**438 passed, 3 skipped** (75s)

降幅来自 `test_guard.py` (~35) + `test_indicators.py` (~10)。
