# 2026-03-17 管理后台视觉优化 + 默认管理员 env 引导

## 本轮目标
1. 优化前端后台页面视觉表现，减少“只有表单和表格”的粗糙感
2. 增加通过 env 自动引导默认管理员账号的能力，便于 dev/test 直接进入后台联调
3. 保持不读取真实 env 文件，仅更新模板与代码逻辑

## 本轮改动
- 更新 `docs/产品需求文档.md`
  - 将“开发/测试环境支持通过 env 自动引导默认管理员账号（邮箱/密码）”纳入产品要求
- 更新 `docs/认证与管理后台规格说明.md`
  - 明确当 `DEFAULT_ADMIN_EMAIL` 与 `DEFAULT_ADMIN_PASSWORD` 存在时，后端启动应确保该账号为 active admin
- 更新配置模型 `backend/src/shared/config.py`
  - 新增：
    - `DEFAULT_ADMIN_EMAIL`
    - `DEFAULT_ADMIN_PASSWORD`
    - `DEFAULT_ADMIN_USERNAME`
- 新增 `backend/src/services/admin_bootstrap.py`
  - 启动时检查默认管理员 env
  - 若账号不存在则创建
  - 若账号已存在则确保：
    - role=admin
    - status=active
    - username 与配置对齐
    - 密码与 env 同步
- 更新 `backend/src/app/main.py`
  - 在应用启动生命周期中执行默认管理员引导
- 新增测试 `backend/tests/unit/test_admin_bootstrap.py`
  - 覆盖“创建管理员”与“提升既有用户并重置密码”两个场景
- 更新 env 模板：
  - `.env.dev-server.example`
  - `.env.example`
- 前端视觉优化：
  - `frontend/src/app/layout.tsx` 增加环境光背景层
  - `frontend/src/components/app-shell.tsx` 调整品牌文案
  - `frontend/src/app/admin/page.tsx` 重写后台 hero，使其更像控制平面首页
  - `frontend/src/app/globals.css` 优化整体背景、卡片层次、头部、状态卡、后台首页质感与小屏表现

## 为什么这样做
- 用户明确反馈“前端页面太丑”，当前页面虽然功能已接通，但视觉层次和产品感不足。
- 用户还要求把管理员账号与默认密码通过 env 配置进 dev/test 流程，这需要后端在启动时具备“确保默认管理员存在”的能力，而不是依赖手工注册后再提权。
- 出于安全考虑，这套默认密码逻辑仅应在开发/测试环境中使用；生产环境不应依赖固定默认密码。

## 验证
- 前端：`cd frontend && npm run build`
- 后端：`backend/.venv/bin/pytest backend/tests/unit/test_admin_bootstrap.py backend/tests/api/test_auth.py backend/tests/api/test_admin_users.py -q`
- 说明：本轮不读取真实 env 文件；如需让 dev 环境立即生效，需要将默认管理员变量写入 `.env.dev-server` 后重新部署。
