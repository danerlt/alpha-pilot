## 为什么做

- 老板发现近期新增的 4 个迁移命名混乱（`d9076875486b_*`、`aa150ff7dee5_*` 等）—— `alembic.ini` 没配 `file_template` 导致 `alembic revision -m` 默认产 hash 前缀
- 老板指示数据库 schema 不要再带外键，关系全部在业务层用 ID 维护，让 schema 更灵活、便于后续多租户拆分

## 实施过程踩到的坑

1. **第一次 autogenerate 误读 SQLite**：`backend/src/shared/config.py` 的 `_ENV_FILE = Path(__file__).parent.parent.parent.parent / ".env"` 在 alembic 跑 env.py 时拿到的 `__file__` 含未规范化的 `..`（来自 `sys.path.insert(0, "..")`），`.parent` 链算回 `backend/migrations/.env` 而不是项目根 `.env`。修正为 `Path(__file__).resolve().parent.parent.parent.parent / ".env"`
2. **autogenerate 不带 server_default**：model 里只用 Python `default=`，autogenerate 不知道，新 init 迁移里 `risk_profiles.max_position_size_pct` 等列没有 server_default，bootstrap INSERT 缺字段报 NotNullViolation。修正：bootstrap INSERT 显式列出所有 NOT NULL 字段
3. **测试有 mock 已删函数**：`tests/unit/test_db_scripts.py` 在 mock `_needs_legacy_stamp` 与 `get_base_settings`，简化 upgrade_db.py 后函数没了，patch 失败。重写测试只验证 `subprocess.run` 被调用一次
