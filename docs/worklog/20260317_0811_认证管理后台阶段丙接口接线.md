# 2026-03-17 Auth/Admin 前端 Phase F：管理页 API 串联

## 本轮目标
依据 `docs/认证与管理后台规格说明.md` 与 `docs/认证与管理后台计划.md`，继续推进前端 Phase F，优先完成：
1. admin 用户管理页接通真实 API
2. admin 符号管理页接通真实 API
3. 保持移动端可用，不依赖宽表格才能操作
4. 避免继续停留在纯前端 mock 登录态，至少让 admin 页面能带真实 token 调后端

## 本轮改动
- 新增 `frontend/src/lib/api.ts`，统一封装 API base、错误解析和带 bearer token 的请求头生成。
- 重写 `frontend/src/lib/auth.ts` 与 `frontend/src/components/auth-provider.tsx`：
  - 登录/注册改为调用真实后端 `/api/auth/login`、`/api/auth/register`
  - 本地仍保存 session，但页面刷新后会自动调用 `/api/auth/me` 校验 token
  - 注册流程对齐后端约束：默认创建普通用户，不再允许前端自行指定 admin 角色
- 更新 `frontend/src/components/auth-card.tsx` 与 `frontend/src/components/route-guard.tsx`：
  - 登录/注册文案对齐真实后端流程
  - 非管理员访问后台时，引导切换管理员账号，而不是再提供“前端自提权”入口
- 更新 `frontend/src/app/admin/page.tsx`：
  - 后台首页从占位导航升级为真实入口页
  - 增加用户管理与符号管理双入口
- 重写 `frontend/src/app/admin/currencies/page.tsx`：
  - 接通 `GET /api/admin/symbols`
  - 支持 `POST /api/admin/symbols` 新增
  - 支持 `PATCH /api/admin/symbols/{id}` 编辑
  - 补充搜索、状态筛选、统一表单、成功/失败提示
  - 桌面端保留表格，移动端切卡片
- 新增 `frontend/src/app/admin/users/page.tsx`：
  - 接通 `GET /api/admin/users`
  - 支持 `PATCH /api/admin/users/{id}` 修改 role/status
  - 桌面端用表格，移动端用卡片 + 下拉选择
- 扩展 `frontend/src/app/globals.css`：
  - 补 admin 表单网格、移动端卡片、状态标签、桌面/移动双视图切换样式

## 这样改的原因
- 之前 admin 页面虽然有入口和骨架，但仍停留在静态数据阶段，无法验证 Phase F 的“前端 admin 视图已真正连通后端 API”。
- 之前登录态是纯 mock，本地邮箱里带 `admin` 就能看到后台入口，这对真实 admin API 完全不成立；必须至少切到后端 JWT + `/auth/me` 校验，后台页面才能正常带 token 访问。
- spec 明确要求 admin 页面在移动端避免宽表格，因此把桌面表格和移动卡片同时保留，避免手机上只能横向滚动才能用。
