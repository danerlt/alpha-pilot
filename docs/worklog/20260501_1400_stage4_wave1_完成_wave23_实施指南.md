## Wave 2/3 待实施 — 详细指南

### Wave 2：成功响应 `Response[T]` 包装

**目标**：所有 router 函数返回值统一包成 `{"success": true, "code": "0", "message": "成功", "data": {...}, "request_id": "..."}` envelope。

**实施步骤**：

1. **在每个 router 函数加 `@api_response()` 装饰器**（spec §4.6.4 已就位）：

```python
# 原状（src/controllers/api/v1/execution/positions.py）
@router.get("")
async def list_positions(db: Session = Depends(get_db)):
    return [PositionRead.model_validate(p) for p in positions]

# 改造后
from src.common.api_response import api_response

@router.get("")
@api_response()
async def list_positions(db: Session = Depends(get_db)):
    return [PositionRead.model_validate(p) for p in positions]
    # 装饰器自动包成 Response[list[PositionRead]]
```

2. **影响 router 文件**（13 个）：所有 `src/controllers/api/v1/{domain}/*.py`

3. **测试期望同步改**：从 `resp.json()["id"]` 改 `resp.json()["data"]["id"]`，**所有 tests/api/* 要改**（约 13 个文件 × N 测试）

### Wave 3：业务异常替换 `HTTPException` → `AppBaseException`

**目标**：业务异常统一 HTTP 200 + body `success:false` + code 区分；HTTP 状态码留给传输层语义（404 路径不存在、401 未授权等真传输异常）。

**HTTPException 现有调用点**（共 14 处）：

| 文件 | 调用次数 | 替换建议 |
|------|---------|---------|
| `risk/commands.py` | 3 | `RiskRejectedException` / `KillSwitchPausedException` / `ServiceException` |
| `risk/risk_events.py` | 2 | `DBException(NOT_FOUND)` / `ServiceException` |
| `system/admin.py` | 4 | `DBException(NOT_FOUND)` / `ParamsException` / `PermissionError` |
| `system/auth.py` | 4 | `AuthError`（业务专属，需在 `errors.py` 加） |
| `system/runtime_config.py` | 1 | `ServiceException` |

**实施模式**：

```python
# 原状
if not position:
    raise HTTPException(status_code=404, detail="Position not found")

# 改造后
from src.common.exception.errors import DBException
from src.common.response.response_code import ErrorCode

if not position:
    raise DBException(error_code=ErrorCode.NOT_FOUND, message="Position not found")
# 全局 handler 自动转成 HTTP 200 + body {"success":false, "code":"400005", "message":"Position not found"}
```

**测试期望同步改**：
- `assert resp.status_code == 404` → `assert resp.status_code == 200`
- 加 `assert resp.json()["success"] is False`
- 加 `assert resp.json()["code"] == "400005"`

### 前端配套升级

**fetch 封装层**（项目里在 `frontend/lib/api.ts` 或类似位置，老板熟悉）：

```ts
// 旧
async function fetchApi(url, opts) {
  const resp = await fetch(url, opts)
  if (!resp.ok) {
    throw new Error(resp.statusText)
  }
  return resp.json()
}

// 新
async function fetchApi(url, opts) {
  const resp = await fetch(url, opts)
  const body = await resp.json()
  if (!body.success) {
    // 业务错误 — 用 code 分类
    const err = new Error(body.message)
    err.code = body.code
    err.requestId = body.request_id
    throw err
  }
  return body.data
}
```
