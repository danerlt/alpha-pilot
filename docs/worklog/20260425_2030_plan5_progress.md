## 交付摘要

### 1. APScheduler 切换 (eee19aa)

加 `Settings.USE_NEW_PIPELINE_WORKER: bool = False` feature flag:
- `false` (默认) 保留 `_strategy_job` / `_monitor_job` 旧 worker, 行为不变
- `true` 启用 `src/workers/scheduler_jobs.py` 新 worker:
  - `new_strategy_pipeline_job` 调 `run_strategy_pipeline_once`,
    自动构造 BinanceAdapter (按 TRADING_MODE) + LLMClient (按 LLM_PROVIDER,
    占位 key 退回 MockLLMClient HOLD-only)
  - `new_position_monitor_job` 调 `run_position_monitor_once`
  - 从 risk_profiles 表读 active 行注入到 Guard
  - KillSwitch 检查保留: paused 时跳过 strategy_pipeline, 但
    position_monitor 保持运行 (SL/TP 必须保护已开仓位)

dev server 准备好之后, env 加 `USE_NEW_PIPELINE_WORKER=true` 一键切到新
worker, 无需修改代码。

### 2. WebSocket 多通道 + catchup (a72fc45)

- `EventShuttle._pubsub` 旁路: 同时 publish 到 Redis Pub/Sub
  `events:<event_type>` 频道 (细粒度) + `trading_events` 频道 (兼容老订阅)
- `redis_subscriber` 升级: `subscribe(trading_events) + psubscribe(events:*)`
  让前端能按事件类型订阅
- 新 `GET /api/events/catchup?since=<event_id>&limit=N`:
  从 `event_outbox` 读 published_at != null 且 event_id > since 的 envelope.
  UUIDv7 时间有序保证字符串比较即时间比较. 上限 500 (spec §3.5).

测试: events_catchup 4 个用例 (full/since 过滤/limit/排除未 published) 全过.
全量回归 **342 passed**.

### 3. 前端 Dashboard preview 页 (2daf850)

`/preview/dashboard` 路由 (与既有 / 并存) 演示 DS Web Shell:
- AppShell 组合 Sidebar (240px+7 导航+引擎状态) + Topbar (60px+风控胶囊+搜索+AUTO)
- 4 个关键指标卡片 (账户权益/今日盈亏/本周盈亏/持仓) 用 DS Stat + tone (pos/neg)
- 权益走势用 DS Spark
- 最近 AI 决策列表用 Pill 标 action/guard/fallback

V0.1 用 mock 数据演示视觉; 接 lib/api 真实数据 + 重做 / 主路由留 V0.2.
旧 / page.tsx (889 行) 不动, 切换平稳.
