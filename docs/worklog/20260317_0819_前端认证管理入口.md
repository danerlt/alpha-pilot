# 2026-03-17 前端 Auth / Admin 可见入口收口

## 本轮目标
依据 `docs/认证与管理后台规格说明.md` 与 `docs/认证与管理后台计划.md`，优先完成一段用户可见的前端改动：
1. 登录页
2. 注册页
3. 登录态存储与顶部入口
4. 管理后台入口占位 / 导航
5. 管理员币种管理页面骨架串联

## 本轮改动
- 对齐前端角色枚举为 `user | admin`，避免继续使用与 spec 不一致的 `trader`。
- 登录 / 注册页补了更明确的文案、密码长度前端校验、`next` 回跳能力。
- 本地登录态继续使用浏览器存储，但在登录后会按角色跳转：普通用户去 `/`，管理员去 `/admin`。
- 顶部导航按角色显示：
  - 普通用户只看到控制台
  - 管理员才看到“管理后台”与后台快捷入口
- 控制台对普通用户隐藏管理员配置改动入口，只保留登录态说明和只读信息提示。
- 管理后台首页补成更明确的导航占位页，串到币种管理骨架页。
- 登录页 / 注册页加上 `Suspense` 包裹，修复 Next.js build 下 `useSearchParams()` 的 prerender 报错。
