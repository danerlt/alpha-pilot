# Task 1 — 建独立 backend/src/schemas/ 顶层目录

## 做了什么
按 spec §3.1 把分散在 controller 文件里的 Pydantic Schema 类抽到顶层目录 `backend/src/schemas/`，扁平结构按域归类。

### 新建文件 (7 个)
- `backend/src/schemas/__init__.py` — 顶层目录说明
- `backend/src/schemas/auth.py` — `UserRegisterCreate` / `AuthLoginCreate`
- `backend/src/schemas/user.py` — `UserUpdate`
- `backend/src/schemas/symbol_config.py` — `SymbolConfigCreate` / `SymbolConfigUpdate`
- `backend/src/schemas/runtime_config.py` — `RuntimeConfigUpdate`
- `backend/src/schemas/command.py` — `CloseAllCreate` / `ClosePositionCreate` / `ResolveBreakerCreate` / `PauseCreate`
- `backend/src/schemas/risk_event.py` — `RiskEventResolveCreate`

### 重命名 (按 spec 命名规则 xxxCreate / xxxUpdate)
| 旧名 | 新名 |
|------|------|
| RegisterRequest | UserRegisterCreate |
| LoginRequest | AuthLoginCreate |
| AdminUserUpdate | UserUpdate |
| CloseAllRequest | CloseAllCreate |
| ClosePositionRequest | ClosePositionCreate |
| ResolveBreakerRequest | ResolveBreakerCreate |
| PauseRequest | PauseCreate |
| ResolveRiskEventRequest | RiskEventResolveCreate |

保留原名 (本就合规)：`RuntimeConfigUpdate`、`SymbolConfigCreate`、`SymbolConfigUpdate`。

### 修改的 controller (5 个)
- `backend/src/controllers/api/v1/system/auth.py`
- `backend/src/controllers/api/v1/system/admin.py`
- `backend/src/controllers/api/v1/system/runtime_config.py`
- `backend/src/controllers/api/v1/risk/commands.py`
- `backend/src/controllers/api/v1/risk/risk_events.py`

每个文件去掉了内联 `class XxxRequest(BaseModel)` 定义，新增 `from src.schemas.<domain> import ...`，函数签名同步换名。

## 为什么做
spec §3.1 要求 Pydantic Schema 集中到顶层 `schemas/`，形成 model→schema→service→controller 的清晰分层；命名统一为 `xxxCreate/xxxUpdate/xxxRead/xxxOut/xxxQuery`。

## 如何验证
```bash
cd backend && ALPHAPILOT_SKIP_SECRET_VALIDATION=1 uv run pytest -q
```
结果：**439 passed, 2 skipped, 5 warnings in 61.90s** — 完全符合红线。

HTTP 响应 JSON 结构未变：所有 controller 响应仍是手写 dict 拼装，schema 只用于入参校验，零行为差。

## 边界
- 仅迁移了 11 个真正的 HTTP 入参 Pydantic 类。
- 未动 `common.pagination.Paginated`、`common.events.BaseEvent`、`core.exchange.types.*`、`services.events.contracts.EventEnvelope`、`services.strategy.proposal.DecisionProposal` 等内部契约 BaseModel — 它们不属于 HTTP 入参/出参 schema，按 spec 不在 `schemas/` 目录范畴。
- 未建空领域文件 (position/trade/order/decision/report/account/event)：当前 controller 全部直接返回 dict 而非 Pydantic Read 类，建空文件无意义；后续 Task 真要补 Read 类时再在该目录创建。
