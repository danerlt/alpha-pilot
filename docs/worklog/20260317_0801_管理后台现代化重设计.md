# 2026-03-17 后台前端现代化重设计（auth + admin）

## 本轮目标
1. 将“页面太丑”作为最高优先级，做一轮明显可感知的现代化升级
2. 统一 login / register / admin 首页 / users / currencies / audit logs 的视觉语言
3. 保留现有功能可用性，不读取真实 env 文件
4. 完成构建验证并记录结果

## 设计决策（基于 ui-ux-pro-max 思路）
- 延续并强化当前深色控制平面方向，统一成一套暗色玻璃卡片系统
- 优先落实：
  - 清晰信息层级
  - 44px+ 可触达按钮与输入
  - 更稳定的卡片、间距、状态胶囊与按钮反馈
  - 桌面表格 + 移动端卡片的双形态一致性
- 避免继续出现“同功能多种视觉风格混搭”“后台像裸表单堆叠”的问题
- Symbol 管理、用户管理、审计日志全部收口到统一 admin 视觉骨架

## 本轮改动

### 1. 全局视觉系统重做
- 更新 `frontend/src/app/globals.css`
  - 补充统一 focus-visible、按钮 hover/disabled、输入 focus ring
  - 主按钮升级为更强渐变与阴影层级
  - ghost button 改为更稳的玻璃态面板按钮
  - 输入框统一圆角、边框、占位色和触达高度
  - 新增 admin 子导航样式、hero split、highlight card、summary pill、action badge 等设计 token/组件样式
  - 新增 auth modern 样式：hero panel、feature card、micro card、移动端栅格收口

### 2. 顶部品牌与 admin 子导航统一
- 更新 `frontend/src/components/app-shell.tsx`
  - 品牌 logo 从 emoji 改为纯视觉几何点核，避免结构性 emoji 图标
  - admin 页面自动显示子导航
- 更新 `frontend/src/components/admin-subnav.tsx`
  - 子导航统一为：后台首页 / Symbol 管理 / 用户管理 / 审计日志

### 3. 登录/注册页面重设计
- 更新 `frontend/src/components/auth-card.tsx`
  - 从“左介绍 + 右表单”的普通表单页，升级为完整控制平面风格 auth 入口
  - 新增 hero 高亮面板、能力特性卡、双摘要 micro card
  - 保留登录/注册原有 API 流程与跳转逻辑
  - 强化注册/登录状态说明与权限边界提示

### 4. Admin 首页现代化升级
- 更新 `frontend/src/app/admin/page.tsx`
  - hero 改为 split layout：主叙事 + highlight card
  - 模块入口统一为 feature cards
  - 首页信息收口到 symbols / users / audit logs 三块主模块
  - 文案上统一使用 symbol 管理语义

### 5. Users 页面重设计
- 更新 `frontend/src/app/admin/users/page.tsx`
  - 新增 hero + highlight card
  - 搜索条改成带 summary pill 的工具条
  - 增加四个指标卡：总用户、活跃、禁用、管理员
  - 保留桌面表格 / 移动卡片双形态，统一记录卡视觉
  - 修正移动端状态标签展示文案

### 6. Currencies 页面收口为 Symbol 管理
- 更新 `frontend/src/app/admin/currencies/page.tsx`
  - 页面标题、CTA、列表标题、搜索文案全部统一为 Symbol 管理语义
  - 新增 hero + highlight card
  - 新增四个指标卡：总 Symbols、启用、停用、当前筛选
  - 编辑器区块文案统一为更现代、更面向移动端的表达

### 7. 审计日志页骨架补齐并接通 API
- 新增 `frontend/src/app/admin/audit-logs/page.tsx`
  - 提供审计日志 hero、筛选工具条、桌面表格、移动卡片骨架
  - 支持按关键词与资源类型筛选
- 更新 `backend/src/app/router.py`
  - 新增 `GET /api/admin/audit-logs?limit=` 列表接口
- 说明
  - 当前是轻量列表版，先解决后台视觉与结构完整性，再逐步补操作人映射、详情抽屉、更多筛选

## 修复的问题
- 修复审计日志页在 Next/TS build 中的 `Set` 迭代类型报错，改为 `Array.from(new Set(...))`
- 修复 admin 体系里缺少统一子导航的问题

- 修复 users 移动卡片中状态标签直接暴露 raw status 文案的问题
