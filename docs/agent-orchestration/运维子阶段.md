# Ops 内部子阶段

## 1. test-design
输入：
- task.md
- plan/*
- implement/summary.md

输出：
- ops/test-plan.md

## 2. test-run
输入：
- ops/test-plan.md
- repo / branch

输出：
- ops/test-report.md

## 3. review
输入：
- plan/*
- implement/*
- ops/test-report.md

输出：
- ops/review-report.md

## 4. pr
输入：
- implement/summary.md
- ops/test-report.md
- ops/review-report.md

输出：
- ops/pr.md

## 5. merge
输入：
- ops/pr.md
- approval / policy if needed

输出：
- ops/merge.md

## 6. deploy
输入：
- ops/merge.md
- target env
- approval if prod

输出：
- ops/deploy.md

## 群播报策略
ops 对外只需要播报：
1. 开始验证与交付
2. 验证未通过并退回
3. 验证通过，PR/merge/deploy 结果摘要
