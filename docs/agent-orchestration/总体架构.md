# alpha-pilot v1 架构

## 角色

### 1. alpha-pilot
- 主控 Agent
- 接收需求 / 状态查询 / 部署指令 / 优先级调整
- 决定走 feature 还是 bug workflow
- 汇总阶段结果到群
- 管理 run 状态和人工审批

### 2. planner
- 更新 docs 中的需求文档
- 明确变动范围
- 给出技术方案
- 定义验收标准 / 任务拆分
- 推荐模型：Claude

### 3. developer
- 实现 feature
- 修复 bug
- 根据 planner 的方案开发代码
- 推荐模型：Claude

### 4. ops
- 生成测试用例
- 执行测试
- review 代码
- 创建 / 提交 PR
- merge PR
- deploy 到 staging / prod
- 推荐模型：Codex

## 核心原则

1. 群聊不是工作上下文，只是阶段广播面板
2. 每个角色有自己的私有上下文
3. 角色之间通过结构化工件交接，不传整段聊天历史
4. 长期状态落地到 run artifacts
5. 只有 alpha-pilot 长期保留压缩后的管理上下文

## 公开播报

仅播报：
- 阶段开始
- 阶段完成
- 阶段阻塞
- 请求决策
- 最终总结

不播报：
- 原始日志
- 大段 diff
- 每次 retry 细节
- 长链内部思考

## 工作流

### Feature
`intake -> plan -> implement -> ops-verify-and-ship -> summary`

ops 内部：
- test-design
- test-run
- review
- pr
- merge
- deploy

### Bug
`intake -> bug-plan -> fix -> ops-verify-and-ship -> summary`

## 上下文层次

### L1 群聊公开上下文
- 只保留阶段摘要

### L2 run 级共享上下文
- task / plan / handoff / reports / final summary

### L3 角色私有上下文
- 当前步骤的工作记忆
- 阶段完成后压缩回 L2
