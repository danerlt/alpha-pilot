# 上下文模型

## 目标

避免多个 bot 长期滚动上下文导致 token 爆炸。

## 设计

### 1. 身份长期存在
- alpha-pilot
- planner
- developer
- ops

### 2. 会话短期存在
- planner: 每次规划一个短会话
- developer: 小任务一个会话；大任务按 story 或子任务拆会话
- ops: 每个子阶段一个短会话

### 3. 工件长期存在
- `task.md`
- `plan.md`
- `scope.json`
- `implementation-summary.md`
- `test-plan.md`
- `test-report.md`
- `review-report.md`
- `pr.md`
- `deploy.md`
- `final-summary.md`

## run 目录建议

```text
runs/<run-id>/
  task.md
  state.json
  plan/
    requirements.md
    design.md
    scope.json
  implement/
    summary.md
    changed-files.json
  ops/
    test-plan.md
    test-report.md
    review-report.md
    pr.md
    deploy.md
  handoff/
    planner-to-developer.yaml
    developer-to-ops.yaml
    ops-to-alpha-pilot.yaml
  public/
    stage-status.md
    final-summary.md
```

## Handoff 结构

```yaml
handoff:
  runId: R-xxxx
  from: planner
  to: developer
  stage: plan
  status: done
  summary: 已完成需求文档与技术方案
  artifacts:
    - plan/requirements.md
    - plan/design.md
    - plan/scope.json
  constraints:
    - 暂不改数据库 schema
  risks:
    - 旧接口兼容性
  nextAction: implement according to plan
```

## 群消息回流规则

默认不回流。

只有以下内容可回流到执行上下文：
1. 用户明确新增约束
2. alpha-pilot 发布的正式调度指令
3. 审批 / 拒绝 / 优先级变更
