# Runner 使用说明

## 配置文件

- `config/agent_runners.json`

当前映射：
- planner -> Claude
- developer -> Claude
- ops -> Codex

## 脚本

- `scripts/run_agent_stage.py`

## 用法

### 生成 planner prompt 与建议命令
```bash
python3 scripts/run_agent_stage.py <run_id> planner
```

### 生成 developer prompt 与建议命令
```bash
python3 scripts/run_agent_stage.py <run_id> developer
```

### 生成 ops prompt 与建议命令
```bash
python3 scripts/run_agent_stage.py <run_id> ops
```

## 当前阶段行为

当前脚本现在会先做三件事：
1. 校验当前 `state.currentStage` 是否允许该角色执行
2. 校验上游关键工件与 handoff 是否已存在，且不是模板占位内容
3. 生成 prompt 文件并输出建议执行命令

已兼容历史命名差异：
- `developer`
- `implementer`（会自动归一到 `developer`）

后续再把真实 Claude/Codex CLI 执行、结果解析、工件写回接上。
