# 下一步实施建议

## 第 1 步：把本地脚手架映射到真实项目目录
确认 alpha-pilot 真正代码仓库位置，并决定：
- 这些文件是直接放入 repo
- 还是作为独立 orchestration 配置仓库存在

## 第 2 步：决定 antfarm 扩展方式
需要确认 antfarm 是否已经支持 `execution.agentId`。
如果没有，需要补执行层，让每个 agent 指定 Claude/Codex。

## 第 3 步：定义 run 创建机制
alpha-pilot 收到新任务后，需要：
- 生成 run id
- 复制 RUN_TEMPLATE
- 写入 task.md / state.json
- 驱动对应 workflow

## 第 4 步：把 handoff 与群播报接起来
当阶段完成时：
- 更新 handoff 文件
- 更新 public/stage-status.md
- 生成一条群播报

## 第 5 步：接 Telegram 配置
用户提供：
- alpha-pilot / planner / developer / ops 的 bot token
- bot username
- 群 chat id
- 哪些 bot 可在群里主动发言

## 第 6 步：做一次端到端演练
从一个简单 feature 开始：
- planner 出文档
- developer 改代码
- ops 出测试/review/PR/deploy 结果
- alpha-pilot 总结
