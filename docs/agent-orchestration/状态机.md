# 状态机

## Run 状态

- `pending`
- `running`
- `blocked`
- `awaiting_approval`
- `retrying`
- `done`
- `failed`

## 角色输出对状态的影响

### planner
- `STATUS: done` -> 进入 implement
- `STATUS: question` -> `awaiting_approval`
- `STATUS: blocked` -> `blocked`

### developer
- `STATUS: done` -> 进入 ops-verify-and-ship
- `STATUS: blocked` -> `blocked`

### ops
- `STATUS: done` -> 进入 summary
- `STATUS: retry` -> `retrying` 并返回 developer
- `STATUS: approval` -> `awaiting_approval`
- `STATUS: blocked` -> `blocked`

### alpha-pilot
- summary 完成后 -> `done`

## Retry 规则

### feature
- ops 发出 `retry` 时，返回 `implement`
- `state.retryFrom = "ops-verify-and-ship"`

### bug
- ops 发出 `retry` 时，返回 `fix`
- `state.retryFrom = "ops-verify-and-ship"`

## 失败条件

满足任一：
- 达到最大重试次数
- 关键配置缺失且无法自动恢复
- 用户明确终止 run
