# Run IO 规则

## 目标

定义 run 在创建、阶段推进、阻塞、完成时如何读写文件。

## 创建输入

alpha-pilot 在收到新任务时，需要输入：
- task title
- task type: feature|bug|deploy
- original request
- requested by
- workflow id

## 创建输出

创建 run 后，至少写入：
- `task.md`
- `state.json`
- `public/stage-status.md`

## 阶段推进写入规则

### planner 完成后
写入：
- `plan/requirements.md`
- `plan/design.md`
- `plan/scope.json`
- `handoff/planner-to-developer.yaml`
- `state.json`
- `public/stage-status.md`

### developer 完成后
写入：
- `implement/summary.md`
- `implement/changed-files.json`
- `handoff/developer-to-ops.yaml`
- `state.json`
- `public/stage-status.md`

### ops 完成后
写入：
- `ops/test-plan.md`
- `ops/test-report.md`
- `ops/review-report.md`
- `ops/pr.md`
- `ops/merge.md`
- `ops/deploy.md`
- `handoff/ops-to-alpha-pilot.yaml`
- `state.json`
- `public/stage-status.md`
- `public/final-summary.md`（如果已完成）

## 阻塞写入规则

任一阶段阻塞时，应更新：
- `state.json.status = blocked` 或 `awaiting_approval`
- `state.json.blockedReason`
- `public/stage-status.md`

## 完成写入规则

最终完成时，应更新：
- `state.json.status = done`
- `state.json.currentStage = done`
- `public/final-summary.md`
