# 产品规则

## 公开交互规则

1. 群里只播报阶段性结果
2. 子角色默认不展开内部执行细节
3. 用户可以直接 @ 子角色，但该约束应写入 run artifacts
4. 群消息默认不作为执行上下文输入
5. 只有用户明确新约束和 alpha-pilot 正式调度指令才回流

## 上下文规则

1. alpha-pilot 维护压缩后的全局上下文
2. planner / developer / ops 默认使用短会话
3. developer 在大任务时按 story/子任务切 session
4. ops 的 test/review/pr/deploy 按子阶段切 session
5. 长期状态必须落地为文件工件

## 交付规则

1. 没有明确批准时，不做 prod deploy
2. staging deploy 可以作为默认交付目标
3. 验证未通过时必须明确打回 developer
4. alpha-pilot 负责对老板汇总最终结果
