# 执行器桥接

## 目标

把角色执行和模型/CLI 绑定起来：

- planner -> Claude
- developer -> Claude
- ops -> Codex

并让这些执行器与 run artifacts / state / handoff / public summary 脚本配合工作。

## 角色到执行器映射

### planner
- 默认执行器：Claude Code CLI
- 用途：需求文档、范围界定、技术方案

### developer
- 默认执行器：Claude Code CLI
- 用途：代码实现 / bug fix

### ops
- 默认执行器：Codex CLI
- 用途：测试、review、PR、merge、deploy

## 推荐调用方式

### Claude
使用：
```bash
claude --permission-mode bypassPermissions --print '...'
```

### Codex
使用：
```bash
codex exec --full-auto '...'
```
或在需要时使用 PTY 模式。

## 当前阶段策略

当前先补最小 runner 草案：
- 生成角色任务 prompt
- 调用对应执行器
- 将结果写回 run artifacts
- 更新 state / handoff / public summary

## 当前边界

- 不在本阶段直接实现完整 Telegram 接线
- 不在本阶段直接接生产部署命令
- 先以 dry-run / prompt 生成 / 本地文件流为主
