# alpha-pilot 汇总规则

## 目标

让 alpha-pilot 从结构化 handoff 与 artifacts 中生成人类可读摘要，而不是复述全部内部过程。

## 输入来源

alpha-pilot 汇总时只应读取：
- `task.md`
- `state.json`
- `handoff/planner-to-developer.yaml`
- `handoff/developer-to-ops.yaml`
- `handoff/ops-to-alpha-pilot.yaml`
- `public/stage-status.md`
- `public/final-summary.md`

## 阶段摘要格式

### 规划阶段
- 需求是否已澄清
- 变动范围是什么
- 主要风险是什么

### 实现阶段
- 完成了哪些改动
- 哪些模块受影响
- 是否已移交 ops

### 交付阶段
- 测试结果
- review 结论
- PR 状态
- merge 状态
- deploy 结果

## 最终总结模板

```text
本次任务已完成：
- 规划：...
- 实现：...
- 测试与 Review：...
- PR / Merge：...
- 部署：...
- 后续事项：...
```

## 禁止行为

- 不要把原始测试日志整段播报到群里
- 不要把内部 retry 细节全部公开
- 不要把所有文件变更逐条念给用户
