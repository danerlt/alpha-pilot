# 最小胶水脚本

## 已有脚本

### 1. `scripts/init_agent_run.py`
初始化 run：
- 复制 `RUN_TEMPLATE`
- 写入 `task.md`
- 写入 `state.json`
- 写入 `public/stage-status.md`

### 2. `scripts/run_agent_stage.py`
最小角色执行桥接脚本：
- 校验当前 `state.currentStage` 是否允许对应角色执行
- 校验上游关键工件与 handoff 是否已存在，且不是模板占位内容
- 生成 prompt 文件
- 根据角色映射输出建议执行命令
- 兼容 `developer` / `implementer` 历史命名差异

### 3. `scripts/update_agent_state.py`
更新 run 状态：
- `status`
- `currentStage`
- `blockedReason`
- `approvalNeeded`
- `retryFrom`

### 4. `scripts/write_handoff.py`
写入 handoff 文件：
- planner -> developer
- developer -> ops
- ops -> alpha-pilot

### 5. `scripts/render_public_status.py`
根据 `state.json` 和 handoff 摘要，生成：
- `public/stage-status.md`
- `public/final-summary.md`

## 作用

这几支脚本让 AlphaPilot 从“只有设计文档”进入到“可以驱动文件状态流”的阶段。
