# 输出契约

## Planner 输出契约

必须包含：
- `STATUS: done|question|blocked`
- `REQUIREMENTS_DOC`
- `DESIGN_DOC`
- `SCOPE_SUMMARY`
- `ACCEPTANCE`
- `RISKS`

## Developer 输出契约

必须包含：
- `STATUS: done|blocked`
- `IMPLEMENTATION_SUMMARY`
- `CHANGED_FILES`
- `TEST_HINTS`

## Ops 输出契约

必须包含：
- `STATUS: done|retry|approval|blocked`
- `TEST_RESULT`
- `REVIEW_RESULT`
- `PR_RESULT`
- `MERGE_RESULT`
- `DEPLOY_RESULT`

当 `STATUS: retry` 时，必须额外包含：
- `ISSUES`

当 `STATUS: approval` 时，必须额外包含：
- `APPROVAL_REQUEST`

## alpha-pilot 输出契约

对群播报时，应尽量压缩为：
- 阶段状态
- 核心结果
- 风险 / 阻塞
- 是否需要老板决策
