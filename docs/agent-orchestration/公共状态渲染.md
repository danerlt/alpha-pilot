# 公开状态渲染规则

## 目标

从 run artifacts 生成适合群里看的阶段播报，而不是直接转发原始工件内容。

## 输入来源

- `state.json`
- `task.md`
- `handoff/*`
- `public/stage-status.md`
- `public/final-summary.md`

## 渲染原则

1. 每次播报只讲一个阶段
2. 先讲状态，再讲结果，再讲风险/阻塞
3. 不转发长日志
4. 不转发长 diff
5. 不复述所有历史过程

## 阶段播报模板

### 规划完成
规划完成：
- 范围：{{scope_summary}}
- 方案：{{design_summary}}
- 风险：{{risk_summary}}

### 实现完成
实现完成：
- 改动：{{implementation_summary}}
- 关键文件：{{changed_files_summary}}
- 下一步：交给 ops 验证与交付

### ops 完成
交付阶段完成：
- 测试：{{test_result}}
- Review：{{review_result}}
- PR/Merge：{{pr_merge_result}}
- 部署：{{deploy_result}}

### 阻塞
当前任务阻塞：{{blocked_reason}}
需要老板决定：{{decision_needed}}

## 最终总结模板

本次任务已完成：
- 规划：{{plan_summary}}
- 实现：{{implementation_summary}}
- 测试与 Review：{{ops_summary}}
- 部署：{{deploy_summary}}
- 后续：{{follow_up}}
