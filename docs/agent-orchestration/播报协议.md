# 群播报协议

## 原则

- 只播报阶段性结果
- 不播原始日志
- 不播长 diff
- 不自动把群播报回灌成执行上下文

## 消息类型

### 1. 阶段开始
示例：
- alpha-pilot：收到新需求，进入规划阶段。
- planner：我开始整理需求文档、变更范围和技术方案。
- developer：我开始按方案实现代码。
- ops：我开始验证、review 和交付流程。

### 2. 阶段完成
示例：
- planner：规划完成，已输出需求文档、变更范围和技术方案。
- developer：实现完成，已交给 ops 验证与交付。
- ops：验证完成，PR 已创建并已部署到 staging。

### 3. 阶段阻塞
示例：
- ops：验证未通过，已退回 developer。主要问题是边界条件处理不足。
- alpha-pilot：当前任务阻塞，需要老板拍板是否允许缩小范围继续。

### 4. 请求决策
示例：
- planner：当前存在两种技术方案，请拍板。
- ops：生产部署前需要批准。

### 5. 最终总结
由 alpha-pilot 发。
