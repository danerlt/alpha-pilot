# AlphaPilot

> 面向数字货币市场的 AI 自主交易系统  
> **Pilot your alpha.**

AlphaPilot 是一个面向 **Binance** 的 **AI 自主数字货币交易系统**。  
它的目标不是只做行情分析或信号提示，而是在 **严格风控** 和 **受限策略框架** 下，完成从市场理解、交易决策、自动执行到持续优化的完整闭环。

AlphaPilot 的核心理念是：

**让 AI 不只是“建议交易”，而是在明确边界内真正参与交易，并通过经验积累持续提升。**

# 项目定位

AlphaPilot 不是一个简单的聊天式交易助手，也不是一个没有约束的自由交易机器人。  
它是一个以 AI 为核心、以风控为边界、以执行为闭环、以学习为增强的自主交易系统。

它希望解决的问题包括：

- 降低人工盯盘和手工执行成本
- 把交易决策变成结构化、可审计的系统输出
- 让 AI 在规则约束下完成自动交易
- 通过历史经验积累，持续优化策略选择和参数配置
- 避免“高频冲动式 AI 下单”带来的失控风险

# 核心特性

## 1. AI 结构化交易决策
系统基于行情数据、技术指标、账户状态和市场状态，输出结构化交易决策，而不是自由文本建议。

例如：

```json
{
  "symbol": "BTCUSDT",
  "timeframe": "1h",
  "action": "OPEN_LONG",
  "confidence": 0.78,
  "entry_type": "MARKET",
  "entry_price": 68250,
  "stop_loss": 67380,
  "take_profit": 69800,
  "position_size_pct": 0.12,
  "strategy_mode": "trend_following",
  "reasoning": [
    "1h趋势向上",
    "价格站上EMA20和EMA50",
    "MACD金叉",
    "成交量温和放大"
  ],
  "risk_note": "波动率上升，仓位控制在12%"
}
```

## 2. Binance 优先
MVP 版本只接入 Binance，优先覆盖：

- BTCUSDT
- ETHUSDT

后续再扩展更多币种。

## 3. 风控优先
在 AlphaPilot 中，AI 不能直接绕过规则下单。  
所有交易决策都必须通过执行守卫和风险引擎的检查。

核心原则：

- AI 负责提出交易动作
- 风控系统负责决定是否允许执行
- 交易所只接受经过批准的指令

## 4. 自动执行闭环
系统支持：

- 自动读取行情和账户状态
- 自动生成交易决策
- 自动校验风险
- 自动下单 / 平仓
- 自动设置止损止盈
- 自动记录日志与交易结果

## 5. 可解释、可审计
每一笔交易都必须回答这些问题：

- 为什么开仓
- 为什么平仓
- 为什么拒绝执行
- 为什么触发熔断
- 为什么调整参数
- 为什么发生回滚

## 6. 受控自我进化
AlphaPilot 支持“越交易越智能”，但这种进化是受控的，而不是无限制自主演化。

系统可以学习和优化的内容包括：

- 策略权重
- 指标参数
- 仓位建议系数
- 市场状态识别阈值
- 提示词中的经验摘要

系统**不能**自我修改的内容包括：

- 最大单笔风险
- 最大总仓位
- 每日最大亏损
- 熔断规则
- 必须止损的约束
- API 异常停机规则

# 系统能力概览

AlphaPilot 聚焦于以下能力：

1. 接入 Binance 行情与账户数据
2. 计算技术指标与市场特征
3. 识别市场状态（Regime）
4. 生成结构化 AI 交易决策
5. 通过执行守卫进行风险校验
6. 自动完成下单、止损、止盈和平仓
7. 监控持仓、收益与风险状态
8. 基于交易经验进行受控优化

# 核心设计原则

## 受限自主
系统允许 AI 自主判断，但不允许 AI 绕过边界。

## 风控高于模型
风险控制永远优先于模型输出。

## 结构化优先
AI 输出必须是结构化、可验证、可追踪的数据，而不是模糊文本。

## 闭环优先
系统不仅要“给建议”，还要“能执行、能监控、能复盘”。

## 学习可控
所有学习必须：

- 先验证
- 再上线
- 可审计
- 可回滚

# 主要模块

## 市场数据层
负责接入 Binance 的行情、K线、成交量、账户和订单信息。

## 指标与特征层
负责计算：

- EMA
- RSI
- MACD
- ATR
- 布林带
- 波动率
- 支撑阻力摘要
- 多周期一致性

## 市场状态识别层
识别当前市场是：

- 趋势上涨
- 趋势下跌
- 震荡整理
- 高波动混乱

## AI 决策层
负责在受限策略框架下输出交易动作：

- OPEN_LONG
- CLOSE_LONG
- HOLD

## 风控与执行守卫层
负责在下单前进行风控检查：

- 仓位检查
- 止损检查
- 风险收益比检查
- 熔断检查
- API 状态检查
- 市场异常检查

## 交易执行层
负责与 Binance API 交互，完成：

- 开仓
- 平仓
- 止损
- 止盈
- 状态同步

## 监控与复盘层
负责展示与记录：

- 账户权益
- 当前持仓
- 收益情况
- 风控事件
- 每笔交易解释
- 每日复盘报告

## 经验与学习层
负责：

- 记录交易经验
- 检索相似案例
- 评估策略表现
- 生成优化候选
- 做 shadow mode 验证
- 自动回滚失败更新
